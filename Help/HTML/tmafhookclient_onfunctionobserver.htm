OnFunctionObserver

 Top  Previous  Next

Event will be fired, if the current TmafHooClient is observing one or more DynamicFunction and in one of those a change occurred. That can be either a new SubHook being added or an existing one deleted from the DFT.

 

Class

TmafHookClient

 

Syntax

 

TFunctionObserverCallBack = procedure(Sender: TObject; nHookID, nSubHookID : Integer; Action: TFunctionObserverAction) Of Object;

 

Remarks

nHookID defines the DynamicFunction that changed. nSubHookID contains the SubHookID of the function that has been added or removed.

Action can either be foaAdd or foaDelete, indicating, what exactly happened to the SubHook.

If the observer is watching the whole DynamicFunction, for each change this event will be fired.

If a SubHook was overwritten (re-routed to another module and function), the event will fire twice, first for the removal of the old SubHook and then for the adding of the new SubHook.
