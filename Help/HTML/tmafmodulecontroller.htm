Modular Application Framework Components

TmafModuleController

 Top  Previous  Next

The TmafModuleController manages code within a module and must be placed in a TDataModule which must be registered in the start-up code within every module (DLL).

The following Properties MUST be set in order to work with the TmafModuleManager :

 

ModuleInfo.ModuleID : Unique ID for every module

ModuleInfo.ModuleName : the name of the DLL file (i.e. Router_UniDAC.dll)

 

If those aren't set the ModuleManager will reject the loading of that module.

Description

Executes a SubHook. Do not call as it is used internally by the file uModuleService.pas

Registers an event from type TOnSubHook for a specific SubHook

UnregisterSubHookEvent

Unregisters an previously registered event

RegisterFormClass

Registers the class of a form for a specific SubHook. It is used to auto-create forms.
