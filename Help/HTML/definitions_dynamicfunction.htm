DynamicFunction

 Top  Previous  Next

A DynamicFunction is basically represented by its ID (HookID) and contains 1 - n SubHooks. These SubHooks do have an order, which can be changed in design- and run-time. The order defines when certain code is executed, very much like a programmer writes his procedures with calls to other procedures or functions.

 

Parameters can be set and results taken during the events BeforeCallRouter, AfterCallRouter, BeforeExecHook and AfterExecHook.

 

If there is no SubHook left inside a DynamicFunction, it will cease to exist. Calling an non-existing DynamicFunction is no error and will always return ERR_NO_ERROR.
