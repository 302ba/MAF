__Add_FunctionObserver

 Top  Previous  Next

Adds an observer to a DynamicFunction to get informed about changes (adding, deleting, disabling or enabling SubHooks).

 

Class

TmafHookClient

 

Syntax

 

function __Add_FunctionObserver(nHookID, nSubHookID: Integer; AReceiver: TComponent = nil): Integer;

 

Remarks

nHookID defines the ID of the DynamicFunction to observe.

nSubHookID defines a SubHook to observe. Is this parameter is 0, the whole DynamicFunction will be observed.

AReceiver can contain a receiver component for the message sent by the TmafHookManager. For application development the parameter can just be left blank, but components should use this to intercept these messages instead of using the event fired.

The result is an error code. If the execution was successful, ERR_NO_ERROR is returned.

 

If the DynamicFunction is changed (through installing or de-installing a module) the event OnFunctionObserverCallBack is fired.

 

The function observer should always be de-installed using __Delete_FunctionObserver to avoid callbacks into already destroyed objects.
