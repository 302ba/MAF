SubHook

 Top  Previous 

A SubHook defines a single function in a module. It is a member to one DynamicFunction only, but its code can be assigned to several SubHooks.

The SubHook uses a local ID to identify itself withion a module. This ID must be unique within the module and should be unique within the DynamicFunction.

Additionally the SubHook has a global unique ID that is used for installation purposes. This ID is managed by the DFT automatically.

If a new module wants to overwrite an existing SubHook with own code, it has to overwrite this unique ID, as it cannot be certain that the function still can be found using the combination HookID, SubHookID and ModuleID (due to the installation of another module before overwriting the same SubHook).

 

 

If the DynamicFunction doesn't exist when the SubHook gets installed, the DynamicFunction will be created.
